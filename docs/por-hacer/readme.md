# Por Hacer (To Do)

Esta carpeta contiene tareas pendientes, mejoras sugeridas y temas que requieren revisión o discusión con el equipo.

## Estructura

Documentos en esta carpeta (por hacer / por implementar):
- [ROADMAP.md](ROADMAP.md) — Hitos y planificación
- [TECH_DEBT.md](TECH_DEBT.md) — Deuda técnica
- [00-Autorizacion-Permisos-Estado-Completo.md](00-Autorizacion-Permisos-Estado-Completo.md) — Auditoría autorización y permisos
- [01-Revision-Validaciones-Cliente-Pedido.md](01-Revision-Validaciones-Cliente-Pedido.md) — Revisión validaciones cliente/pedido
- [00-POR-IMPLEMENTAR-README.md](00-POR-IMPLEMENTAR-README.md) — Redirecciones desde contenido reubicado

## Propósito

Esta carpeta sirve como:
- **Lista de tareas pendientes** que requieren atención
- **Documentación de mejoras sugeridas** que necesitan revisión
- **Registro de temas** que requieren discusión con stakeholders
- **Tracking de decisiones** pendientes

## Cómo usar esta carpeta

1. **Crear nuevos documentos** cuando se identifiquen temas que requieren revisión
2. **Numerar los documentos** secuencialmente (01, 02, 03...)
3. **Incluir información relevante:**
   - Problema identificado
   - Estado actual
   - Recomendaciones
   - Acciones sugeridas
   - Referencias técnicas

4. **Actualizar el estado** cuando se complete o se tome una decisión

## Estados sugeridos

- **Pendiente** - Aún no se ha abordado
- **En revisión** - Está siendo analizado
- **En discusión** - Requiere decisión de stakeholders
- **Aprobado** - Se ha decidido implementar
- **Rechazado** - Se ha decidido no implementar
- **Completado** - Ya se ha implementado o resuelto
